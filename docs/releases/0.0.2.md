# Mac Bastion 0.0.2

Maintenance release with reliability fixes for tunnel lifecycle and config includes.

## Downloads

- `MacBastionMenu-0.0.2-macos-universal.zip`: macOS menu bar app.
- `mbastion-0.0.2-macos-universal.tar.gz`: CLI executable and runtime library.
- `SHA256SUMS.txt`: artifact checksums.

## Fixes

- **Config includes**: diamond includes (the same file reachable through two
  separate include paths) no longer load twice and duplicate profiles. Include
  cycle detection is preserved.
- **`stop-all`**: now also stops orphaned tunnels whose profiles were removed
  from config, instead of leaving them running indefinitely.
- **Safer stop**: before signalling a recorded PID, `stop` verifies the live
  process is still the ssh tunnel it launched, so a recycled PID can no longer
  cause an unrelated process (shell, editor, ...) to be killed. When the process
  cannot be verified, it is left running rather than abandoned.
- **Runtime record decoding**: fixed an ISO8601 date-strategy mismatch when
  decoding runtime records.

## Notes

- Binaries are now universal (Apple Silicon `arm64` + Intel `x86_64`), so the
  CLI and menu bar app run natively on both architectures.
- Runtime records gain an optional `executable` field; records written by 0.0.1
  remain fully compatible in both directions.
- Adds a Korean README and expands the English README with feature, profile,
  and CLI reference tables.
- The app is ad-hoc signed, not Apple Developer ID notarized yet. On first
  launch macOS may require right-clicking the app and choosing Open. If
  Gatekeeper still blocks a downloaded zip, run:

```sh
xattr -dr com.apple.quarantine /Applications/MacBastionMenu.app
```
